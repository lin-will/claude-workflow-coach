## Build Log — Entry 002
*Date: May 6, 2026*

**Project:** Claude Workflow Coach — a web app that coaches non-engineers through professional Claude Code workflows, from dev environment setup to agentic prompting and MCP integration.

### Decisions locked this session

**Problem statement:**
Non-engineers struggle to use Claude Code as a real development tool to enhance their workflows because they're not experienced with professional development workflows and the existing possibilities of Claude Code, so they continue to use AI as a search engine and basic code generator instead of a comprehensive force multiplier.

**Solution statement:**
The Claude Workflow Coach guides non-engineers through professional Claude Code workflows — from dev environment setup to agentic prompting and MCP integration — so they can use AI as a true development partner instead of a basic code generator.

**Scope decision:**
Git and GitHub workflows are explicitly out of scope for the app. They are a separate personal learning track for the PM only — not a feature.

**Two parallel tracks going forward:**
- 🏗️ Build track — PRD, architecture, app
- 🎓 Learning track — GitHub, Claude Code, dev environment setup

Learning track confusion = primary research and potential feature insight.

### PM skills practiced
- Problem before solution
- Scope decisions in writing
- Being the user as primary research
- Pressure-testing your own statements
- Separating problem statement from solution statement

### Key insight
PMs are almost never domain experts. Their job is to feel the user's pain, ask good questions, find the experts, and reason from real problems — not to know everything upfront. Your personal learning gaps are your product research.
